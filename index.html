My Project Idea

My idea for a project is a text-based combat-focused Role-Playing Game (RPG) made in Python using the Pathfinder Tabletop Role-Playing Game (TTRPG) system and involving randomly generated, rather than pre-scripted, combat, and narrative scenarios and sequences.

Pathfinder is a hugely fun system with a lot of complexity that allows for huge levels of customisability and leads to some considerably enjoyable emergent gameplay. Pathfinder does have a video game version called Kingmaker; however, this has widely been criticised (see: countless user reviews) for its underwhelming and often dull writing, as well as its extreme lack of faith towards the tabletop rules and system. On top of this, most mods of the game that make improvements are no longer supported, such as the Turn-Based Combat mod which restored the original turn system.

Additionally, an issue with pre-written story pathways aside from their quality is their repetition. The fun of TTRPGs is the prospect of experiencing completely new things each time you play, as the boundaries of a TTRPG narrative are endless. Being forced into playing through the same storyline over and over, as seen in Kingmaker as well as essentially any pre-written TTRPG video game adaptation, takes the fun out of it to an extreme degree.

Conversely, randomly generated RPGs such as aidungeon.io do provide enjoyable and unique experiences, but they do not use RPG systems, and the user only manipulates the direction of a random narrative’s algorithm rather than being presented with a set of defined limits and options that make character-driven role-playing so satisfying. A randomly generated narrative where a player can act using the true Pathfinder ruleset would serve as an enjoyable alternative to Kingmaker by doing everything it does wrong right.

As well as wishing to create a product that is enjoyable and accurate to what fans of the original system want, another purpose of this project is to assist me as a programmer by serving as a major undertaking through which to grow my skills in Python. This is a language that I have recently started learning and must gain a high level of competence at to be considered for many development positions due to its ubiquitousness.

The product itself will be a Python program with the ruleset of Pathfinder built in. This includes distance matrices, combat systems and interactions, races and classes, skills and spells, and all the information and statistics about these parts of the game. The scope of the inclusions would not include the entirety of the system (as that would require a significantly larger development team), but just the core ruleset without any of the (many, many) additions to the system.

The program would use this information to assign information and parameters to objects (characters) within the game. The program will allow a player to create and fully customise a Pathfinder character – choosing a level, name, class, background, alignment, stats, skills, feats, etc. – and manipulate that character within the game until either the character is killed, or the game is otherwise ended.

The player, upon creating a character, will have the option to select some parameters for the environment in which the character will initialise. This will primarily include a setting, such as a castle, a forest, a city, a ship, etc. which would limit the set of encounter possibilities to options broadly related to this setting. For example, upon choosing a forest environment, the game will choose from a list of possible characters with the ‘forest’ value (i.e. goblins, gnomes, gripplis, etc.) and choose them for the encounter as either enemies or interactable non-player characters (NPCs).

A typical game will run through a short scenario within the selected setting comprised of some small number of encounters with enemies and/or other NPCs. Upon the player character’s survival, the game will come to a close with either the option to close the campaign permanently or continue with the same character onto a new setting.

In-game, a player will be able to see a range of information and options for interacting with the game world. The program will show a column listing all information about the player character on the left side of the UI.

A column serving as a log for any relevant information will be shown on the right side of the UI. This will allow the player to input text at will so the player does not have to tab out of the application or write physical notes if there is some mechanic, etc. within the game that requires the player to check some information.

The main part of the program will be the log of scenario text. This will describe to the player what is happening within the game, what characters say and do, including the effects of the player character’s actions.

The player will be able to choose action types from a dropdown at the bottom of the UI. Action types include interact, speak, skill, feat, weapon, defence, spell, etc. and after being selected, the dropdown to the right will give specific options for that selected action type. Options might include ‘attack with sword’ after choosing ‘weapon’ or ‘taunt your enemy’ after selecting ‘speak’.

To complete this project, all that is needed is the Pathfinder SRD website, Python 3.x installed, and an Integrated Development Environment (IDE) such as Visual Studio Code that support and include features for programming in Python. Additionally, a GUI framework for Python could be used, such as wxPython which is simple and open-source, ideal for the basic UI of this project.

Knowledge of Python is the main condition for making this project a success, and to gain the required knowledge, experience working with the language must be built up. Working on quick, smaller, related projects before this one begins such as building a similar GUI, making scripts that generate random strings of text from option lists, making scripts wherein two characters with stat blocks make decisions during combat with each other, etc. These will work as good preparatory exercises and may also serve as the basis for some parts of the code in the project itself.

The successful completion of this project will have two outcomes. The first will be that players of the Pathfinder TTRPG system who would like to interact with it in a single player setting and do not wish to use Kingmaker will have an option to have that experience available to them with this program.

The program should have a list of features that, while not exhaustive, should contain a significant enough portion of the system’s ruleset, as well as a list of possible settings, characters, actions, etc. as expansive enough that the game will have a high level of replay value.

The second outcome will be that I will have grown as a Python programmer, with a quality program under my belt to show potential employers my value as a programmer and to use as a steppingstone to move onto larger projects and improve my skills even more so.
